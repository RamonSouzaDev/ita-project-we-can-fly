🛠️ Task 1: Configure Environment & Account

1. Open the Google Cloud Console with the lab credentials and open Cloud Shell (the terminal icon at the top right).

2. Copy, paste, and run these commands one by one to enable the Cloud AI Companion API and grant IAM roles:

PROJECT_ID=$(gcloud config get-value project)
echo "PROJECT_ID=${PROJECT_ID}"

USER=$(gcloud config get-value account 2> /dev/null)
echo "USER=${USER}"

gcloud services enable cloudaicompanion.googleapis.com --project ${PROJECT_ID}
gcloud projects add-iam-policy-binding ${PROJECT_ID} --member user:${USER} --role=roles/cloudaicompanion.user
gcloud projects add-iam-policy-binding ${PROJECT_ID} --member user:${USER} --role=roles/serviceusage.serviceUsageViewer

💡 Tip: Click "Check my progress" in the lab instructions window once this is done.

🗄️ Task 2: Create Dataset & Enable Gemini

1. In the Cloud Console menu, go to BigQuery.

2. In the Explorer panel, click the three dots next to your project ID and select Create dataset.

3. Name the dataset bqml_tutorial and click Create dataset.

4. Click the Gemini Cloud Assist icon in the top toolbar to enable it.

5. In the query editor, click the Gemini icon and check all options (Auto completion, Auto generation, Explanation).

📖 Task 4: Explain SQL Queries

1. Click SQL QUERY to open a new tab.

2. Paste this query:

SELECT u.id as user_id, u.first_name, u.last_name, avg(oi.sale_price) as avg_sale_price   
FROM `bigquery-public-data.thelook_ecommerce.users` as u   
JOIN `bigquery-public-data.thelook_ecommerce.order_items` as oi   
ON u.id = oi.user_id   
GROUP BY 1,2,3   
ORDER BY avg_sale_price DESC   
LIMIT 10

3. Highlight all the text, right-click, and select Explain current selection. Gemini will explain it on the side.

⚡ Task 5: Generate a SQL query

1. Open a new query tab (+). Click the generate SQL with Gemini magic wand icon.

2. Paste this prompt and click Generate:

# Generate a BigQuery SQL query to calculate the daily total sales for each product. The query should join the order_items table with the products table, both from the bigquery-public-data.thelook_ecommerce dataset, using a LEFT JOIN on the product ID. The final output must include the order date, product ID, product name, and the total sales, rounded to two decimal places. Use descriptive aliases for the tables and group by the column names rather than their positions. Please ensure the results are ordered with the highest total sales at the top.

3. Once Gemini returns the query, click Insert and then Run.

🔮 Task 6: Build a Forecasting Model

1. Open a new query tab, paste this SQL to create the Machine Learning model, and click Run:

CREATE MODEL bqml_tutorial.sales_forecasting_model
OPTIONS(MODEL_TYPE='ARIMA_PLUS',
time_series_timestamp_col='date_col',
time_series_data_col='total_sales',
time_series_id_col='product_id') AS
SELECT sum(sale_price) as total_sales,
DATE(created_at) as date_col,
product_id
FROM `bigquery-public-data.thelook_ecommerce.order_items`
AS t1
INNER JOIN `bigquery-public-data.thelook_ecommerce.products`
AS t2
ON t1.product_id = t2.id
GROUP BY 2, 3;
Wait for the model to finish training (it may take ~10 minutes).

2. Once finishing, open a new query tab and click the magic wand again (generate SQL with Gemini) and use this prompt:

# Use sales_forecasting_model from the bqml_tutorial dataset in my project to generate a forecast and return all the resulting data.

3. Insert the suggested query into the editor and click Run.
